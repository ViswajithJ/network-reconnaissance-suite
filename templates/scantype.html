This scan type sends SYN packets to the target ports and analyzes the responses to determine if the ports are open, closed, or filtered. It's stealthy and efficient, as it doesn't complete the TCP handshake.

In an XMAS scan, the scanner sends packets with the FIN, URG, and PUSH flags set, effectively lighting up the target's ports like a Christmas tree. It's used to detect open ports on the target system.

Online Resources:
  • CAPEC-303: TCP Xmas Scan
UDP scanning involves sending UDP packets to target ports and analyzing the responses. Since UDP is connectionless, the scanner doesn't expect acknowledgments and determines open ports based on responses or lack thereof.

Online Resources:
  • CAPEC-308: UDP Scan
In a FIN scan, the scanner sends packets with only the FIN flag set. It's used to determine if ports are open or closed based on how the target's system responds to these packets.

Online Resources:
  • CAPEC-302: TCP FIN Scan
This scan type sends packets with only the ACK flag set. It's primarily used to determine if a firewall is present and filtering traffic based on ACK responses.

Online Resources:
  • Nmap's ACK scan: A Stealthy Port Scanning
In a NULL scan, the scanner sends packets with no flags set (NULL packets) to the target ports. It relies on how the target system responds to these packets to determine port status.
